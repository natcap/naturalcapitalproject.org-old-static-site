MAFE-T (Mapping Alternatives For Equivalents - Terrestrial) enables users to estimate the impacts of development activities on terrestrial ecosystems and several of the services they provide, and then to select offsets to efficiently mitigate losses. MAFE-T tracks how people are affected by the environmental impacts of development and mitigation activities, making the consequences of development more transparent and enabling mitigation portfolios to be designed in a way that maintains or restores environmental benefits in a socially equitable manner.

MAFE-T was developed to support offset portfolio selection in Colombia based on its national offset policy and manual. The software tool includes by default all the data necessary to run analyses anywhere in Colombia. The only required user-provided data input is a map of the proposed development site whose impacts should be mitigated.

Static Data: Colombia

MAFE-T uses an internal system of precomputed modeling information to efficiently estimate the impacts of a proposed development site or a possible offset parcel. These data can be either downloaded as part of the installation process or else downloaded here and then accessed from your local computer during installation.

These static data are precomputed for the country of Colombia. Note that if you do not choose to use these static data, you will need to use the static map generators provided with the MAFE tool before you will be able to run MAFE itself. The sample data zipfile (download) includes some example static data for your reference.

Release History

  • MAFE-T 0.5.1 was released on October 13, 2014. It includes a number of bugfixes to the installer and adds some validation to the MAFE-T user interface.
  • MAFE-T 0.5.0 was released on October 9, 2014. This is the initial public release.

System Requirements

  • Microsoft Windows. Tested on Microsoft Windows 7, but may work on previous versions.
  • It is helpful, but not required, to have a GIS tool like ArcGIS or Quantum GIS to explore the results.
