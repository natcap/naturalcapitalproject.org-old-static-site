Release History

  • RIOS 1.0.0b3 was released on June 4, 2013. It is a pubic beta release and is likely to get upgrades shortly after feedback from users.
  • RIOS 0.4.2 was released on November 26, 2012. It is an alpha release and should be used for demonstration purposes only. It supplants all previous versions of RIOS.
  • RIOS 0.3.0 was released on July 31, 2012. It is an alpha release and should be used for demonstration purposes only. It supplants all previous versions of RIOS. Changes in this release include:
    • Core RIOS computational overhaul to address memory issues, runtime complexity and future InVEST integration.
    • Added user defined activities.
    • RIOS outputs includes transition maps.
    • Sediment valuation tradeoff framework; sans the actual sediment model.
    • Core InVEST 3.0 sedimentation model development.
    • Calculate a minimal set of User LULC -> Transitions for the sediment model input table so that users only need to handle a subset of values.
    • Updated General LULC types and coefficients.
    • Over 30 bugs fixed (as documented here)
    • A handful of usability enhancements including, but not limited to, better user interface validation, file filtering, and saving of all RIOS inputs from the user's previous run.
  • RIOS (formerly WATER) 0.2.2 was released on June 20, 2012. It is a major release addressing critical bug fixes as well as performance issues. It additionally updates the sample dataset to Blanco watershed. It is an unstable alpha release and should be used for demonstration purposes only.
  • WATER 0.1.1 was released on April 13, 2012. It is a minor release addressing minor performance issues and usability issues. It additionally updates the sample dataset to the entire (but downsampled) Monterrey Water Fund. It is an unstable alpha release and should be used for demonstration purposes only.
  • WATER 0.1.0 was released on April 12, 2012. It is the first release of the Latin America Water Funds Prioritization tool. This release is an unstable alpha release and should be used for demonstration purposes only.

System Requirements

  • Microsoft Windows. Tested on Microsoft Windows 7, but may work on previous versions.
  • It is helpful, but not required, to have a GIS tool like ArcGIS or Quantum GIS to explore the results.

Support

The tool is under heavy development, so user support is limited. Users are encouraged to submit bug reports directly to our issue tracker. Reports can also be directly emailed to richsharp@stanford.edu.
